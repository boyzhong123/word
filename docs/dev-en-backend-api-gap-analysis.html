当前欠缺的接口与字段

下面这些内容会直接影响页面完整度或真实数据接入。

后端已提供，页面可以继续补充

后端给出的接口比当前页面更多。下面这些能力可以补成页面、抽屉或操作入口。

建议先做的三个补充页面
邮箱验证、忘记 / 重置密码、Webhook 管理。它们都有现成后端接口，页面工作量相对可控，也能明显提高海外开发者门户的完整度。
无需新增页面的接口
POST /api/billing/webhook 属于 PayPal 服务端回调；GET /api/billing/summary 和 GET /api/billing/balance 应直接接入 Overview / Billing，不需要新页面。

优先级建议

先统一产品口径，再接主链路，最后补完整度。

P0 · 先确认

统一海外计费模型

确认账户共享钱包、PayPal-only、调用单价、充值最低金额、充值预设、试用包数量和金额单位。

P1 · 主链路

让 Overview / Billing / Keys / Usage 使用真实数据

补钱包汇总、Usage 明细、交易 envelope、Keys 字段和四维 limits，并把当前本地 mock 接线替换掉。

P2 · 完整度

补项目与账号体验

新增 Projects、头像上传、低余额阈值，并补邮箱验证、重置密码和 Webhook 页面。
